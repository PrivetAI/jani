# Output Patterns

Patterns for skills that produce specific output formats or quality standards.

## Template Pattern

For consistent output structure:

```markdown
## Output Template

```
[HEADER]
Title: {title}
Date: {date}

[BODY]
{content}

[FOOTER]  
Generated by {skill_name}
```

Place templates in `assets/` for complex formats.
```

## Example-Driven Pattern

For output matching specific style:

```markdown
## Output Style

Match this example:

> **Input**: Calculate 2+2
> **Output**: The result is 4.

Key characteristics:
- Direct answer first
- No unnecessary preamble
- Concise sentence structure
```

Show 2-3 examples covering edge cases.

## Quality Checklist

For outputs requiring validation:

```markdown
## Quality Checks

Before finalizing:
- [ ] All required sections present
- [ ] No placeholder text (TODO, TBD, etc.)
- [ ] Formatting matches specification
- [ ] Links/references validated
```

## Format Specifications

For structured outputs (JSON, XML, etc.):

```markdown
## Output Format

```json
{
  "status": "success|error",
  "data": { ... },
  "metadata": {
    "generated_at": "ISO-8601",
    "version": "1.0"
  }
}
```

Required fields: `status`, `data`
Optional fields: `metadata`
```

Include schema or example. Note required vs optional.

## Progressive Output

For streaming/incremental output:

```markdown
## Output Strategy

1. Start with structure/outline
2. Fill sections incrementally  
3. Signal progress: "Generating section 2/5..."
4. Summarize on completion
```

## File Output

For skills generating files:

```markdown
## File Naming

Pattern: `{date}_{descriptor}.{ext}`
Example: `2024-01-15_report.pdf`

## File Location

Save to: `./output/` (create if missing)
```
